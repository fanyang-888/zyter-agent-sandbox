=== SYMPHONY — AI Orchestration Control Plane ===

Symphony is Zyter's AI orchestration layer. It does not execute tasks directly;
it coordinates agents, systems, and humans across complex workflows.

Key specs:
- 42 governed meta-agents
- Model-agnostic (Claude, OpenAI, Gemini, open-source)
- 8 weeks to production (reference: CMS WISeR implementation)
- 80-90% auto-approval rates for prior authorization workflows
- Full audit trail on every decision
- Learning mechanism: prompt tuning + model selection testing (not weight fine-tuning)
- Human-in-the-loop: confidence thresholds adjustable; decreases over time as trust builds

Symphony Architecture (4 layers):
1. Experience & Outcomes layer — user-facing interfaces and outcome measurement
2. Control Plane — orchestration, governance, audit
3. Agentic Runtime — agent execution and tool use
4. Enterprise Integration — connectors to EHR, payer systems, databases

=== PRAXIS — Workflow & Outcomes Engine ===

Praxis is Zyter's domain-specific workflow execution engine. It implements the
SENSE → THINK → ACT → LEARN loop for clinical and operational tasks.

Current Praxis modules (all external-facing, sold to payers/providers):
- Utilization Management (UM): Prior authorization, clinical guideline matching
- Care Management (CM): Post-discharge support, patient coordination
- Claims Processing
- Clinical Encounter support
- PA Coordination

Key differentiators:
- GraphRAG retrieval over Zyter System Graph (not flat semantic search)
- MCG / InterQual clinical guideline integration
- LoRA fine-tuned adapters per clinical task
- RLHF signal from embedded domain experts
- Data flywheel: every workflow outcome becomes training data

=== TRUCARE PHM — Population Health Management ===

TruCare is Zyter's population health management platform.
Serves state Medicaid programs, Medicare Advantage plans, multi-national carriers.

=== SMARTSTART — Deployment Accelerator ===

SmartStart is Zyter's pre-built workflow template library for external clients.
Clients use SmartStart templates to go live faster without building from scratch.
Internal equivalent: none yet (this project builds it).

=== ZYTER SYSTEM GRAPH ===

Proprietary knowledge graph analogous to Palantir Foundry.
Maps entities (patients, procedures, guidelines, decisions) and their relationships.
Every workflow execution writes back to the graph — creates persistent episodic memory.
Ontology updates via dynamic schema mutation (not retraining).
